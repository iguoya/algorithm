# 算法思想

  1. 在定义中（直接或者间接）调用自身

  2. 复杂问题转化为规模较小的相似问题

  3. 少量代码可以描述复杂的计算过程，减少了代码量

# 代码结构

  1. 边界条件，递归终止
  2. 递归关系式（原问题如何分解为子问题的？）

# 思路

  1. 降低问题规模
  2. 求解最小子问题
